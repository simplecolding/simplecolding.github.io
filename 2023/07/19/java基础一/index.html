java基础一

Created2023-07-19|Updated2023-07-20|java
|Post View:

基础知识

  • jvm：java虚拟机，运行java程序
  • 核心类库：预先写好的程序，轮子
  • JRE：java运行环境
  • JDK：java开发工具包

jvm + 核心类库=JRE

JRE + 开发工具（javac，java）=JDK

字面量

数据在程序中的书写格式

变量

作用：存储数据
目的：扩展性

关键字和标识符

  • 关键字不能用来做类名和变量名
  • 标志符就是名字：由数字、字母、下划线和美元符组成，不能以数字开头，不能是关键字，区分大小写

进制

  • 2进制： 0B or 0b
  • 8进制：0
  • 16进制：0x or 0X

基本数据类型

4大类3种

整形

byte,short,int,long

浮点型

float,double

字符型

char

布尔型

boolean(一个字节)

类型转换

  • 自动类型转换：类型范围小的变量可以直接赋值给类型范围大的变量
  • 表达式的自动类型转换：小范围类型的变量，会自动转换成表达式总较大范围的变量类型；表达式中的byte、short、char会转化为int参与运算
  • 强制类型转换：类型1 变量1 = （类型2）变量2

分支结构

if和switch

  • if适合做条件是区间判断的
  • switch适合做条件是值比较的

注意：

  1. switch表达式类型只能是byte，short，int，char，jdk5支持枚举，jdk7支持String；不支持double，float，long。
  2. case给出的值不允许重复，且只能是字面量，不能是变量
  3. 不要忘记break

数组

静态数组

  • type[] name === type name[]；完整写法：type[] name = new type[]{}
  • 数组变量名中存储的是数组在内存中的地址，数组是一种引用数据类型

    初始化

  • 定义时赋值
  • 动态初始化

java内存分配

方法区

字节码文件先加载到这里

栈

方法运行时所进入的内存，变量也在这里

堆

new出来的东西会在这块内存中开辟空间并产生地址

方法

格式

修饰符 返回值类型 方法名（形参列表）{

}

参数传递机制

  • 都是值传递，传输的是实参变量中存储的值的副本
  • 引用类型的参数传递，传递的是类型的地址

方法重载

  • 一个类中，出现多个方法的名称相同，但是他们的形参列表是不同的，那么这些方法就被成为方法重载了（其他内容不管，如：修饰符，返回值类型是否一样都无所谓）
  • 形参列表不同指的是，形参的个数、类型、顺序不同
  • 调用方法的时候会通过参数的不同来区分调用的是那个方法

面向对象

  • 对象本质上是一种特殊的数据结构
  • 一个代码文件中，可以写多个class，但是只有一个能用public修饰，且public修饰的class必须是该代码的文件名
  • 堆内存中的对象，没有变量引用时，就会被判定为垃圾，通过java中的垃圾回收机制自动回收
  • 每一个类的实例，都会有一个字段指向方法区的类地址

this

this是一个变量，可以在方法中拿到当前对象

目的

用来解决变量名称冲突问题

this如何拿到调用它的对象

对象实例在调用类方法的时候，会将自身的地址传递给方法区的类，方法区的的类用this接收

构造器

  • 创建对象时，对象会去调用构造器，同时完成对成员变量的初始化赋值
  • 若不写构造器，会自动生成一个无参构造器
  • 有参构造器存在就不会生成无参构造器了，需要自己手写

封装

把要处理的数据以及这些数据的方法设计到一个对象中去

实体类JavaBean

  • 一个用来保存数据的类，可以用来创建对象，保存某个事物的数据
  • 成员变量必须私有，且要为他们提供get、set方法，必须有无参数构造器
  • 数据和数据的业务处理相分离
